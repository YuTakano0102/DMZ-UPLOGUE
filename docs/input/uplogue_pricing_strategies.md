# Uplogue -- 収益モデルと料金プラン提案

**サービス概要:**\
写真の位置情報と時刻情報から旅行記録を自動生成する旅行ライフログアプリ

**現状の仮定:**\
競合は月額¥1,000前後のサブスクリプションモデル。まずは市場シェア拡大を優先。

**収益目標:**\
1ユーザーあたり年間¥2,400（約¥200/月相当）の平均収益を目指す。

------------------------------------------------------------------------

## 戦略1：フリーミアム＋軽課金サブスク

**モデル概要:** 基本無料＋低価格サブスクリプション\
**具体的プラン案:**\
無料：旅行3件/月まで生成・標準画質保存\
Plus ¥300/月：無制限生成・高画質保存・年表表示・共有デザイン拡張

この戦略は低価格により参入障壁を下げ大量獲得を狙える。特に「とりあえず使う」行動と相性が良く口コミが起きやすい。一方でARPUが低く、課金率が想定以下だとインフラ費を回収しにくいリスクがある。

------------------------------------------------------------------------

## 戦略2：生成クレジット従量課金

**モデル概要:** 使用量ベース課金\
**具体的プラン案:**\
無料：月1旅行まで\
追加生成：¥120/旅行\
まとめパック：10回 ¥1,000

利用頻度に応じた支払いとなるためライト層も心理的に始めやすく、ヘビーユーザーから収益を得られる。収益とコストの連動性が高いのが利点。ただし支払いタイミングが分散し、継続収益の予測が難しくなる。

------------------------------------------------------------------------

## 戦略3：ライフログ年額モデル

**モデル概要:** 年額サブスクリプション（記録保管価値課金）\
**具体的プラン案:**\
年額 ¥2,400：無制限生成・年次アルバム・バックアップ保証\
月額なし（年額のみ）

「人生の記録」としての価値に課金することで価格納得性を高め、長期利用を促進できる。解約率が低く安定収益を作りやすい。一方で初期導入ハードルが高く、PMF前は転換率が低くなる可能性がある。
