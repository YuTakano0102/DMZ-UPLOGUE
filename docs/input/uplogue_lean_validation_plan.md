# Uplogue -- リーン検証プラン

**ビジネスアイデア要約:**\
写真の位置情報と時刻情報から旅行の行程を自動生成する旅行ライフログアプリ。差別化ポイントは"操作不要で記録が完成する"点。

**未検証の仮説/リスク要素:** 1.
ユーザーは「自動生成された旅行記録」に価値を感じるか 2.
旅行後に継続的に利用される習慣サービスになるか 3.
生成コストに見合う収益モデルが成立するか

------------------------------------------------------------------------

## 仮説1: 自動生成記録の価値認知

**検証方法** -
手動運用MVP：写真をユーザーから受け取り、スクリプトでタイムライン生成して返却 -
LP＋サンプル表示で登録意向テスト

**見るべき指標** - 生成後の閲覧完了率 - 「もう一度使いたい」回答率 -
シェア率

**結果に応じたアクション** - 高評価：プロダクト開発へ進む -
低評価：出力形式（地図中心 or ストーリー中心）を変更

------------------------------------------------------------------------

## 仮説2: 継続利用（習慣化）

**検証方法** - 初回旅行生成後、2週間後にリマインド送信 -
次の旅行で再アップロードするか観察

**見るべき指標** - 再利用率 - 次回旅行時のアップロード率 -
30日後アクティブ率

**結果に応じたアクション** - 高継続：ライフログ機能強化 -
低継続：旅行中価値（リアルタイム地図など）へピボット

------------------------------------------------------------------------

## 仮説3: 収益成立性

**検証方法** - 料金アンケート（¥100 / ¥300 / ¥500） -
有料機能をモックで提示し仮決済テスト

**見るべき指標** - 支払い意向率 - 価格弾力性 - 1ユーザーあたり想定LTV

**結果に応じたアクション** - 支払い意向高：サブスク開始 -
低：従量課金またはB2Bデータ販売へ変更
