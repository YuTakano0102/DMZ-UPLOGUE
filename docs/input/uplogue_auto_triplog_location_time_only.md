# Uplogue（絞り込み版）--- 複数写真の「位置情報×時刻情報」から旅行記録を自動作成

## 対象となる顧客課題

旅行後に写真は大量に残るが、**時系列整理・場所の紐づけ・文章化が面倒**で、結果として記録が放置される。

------------------------------------------------------------------------

## ソリューションのコアコンセプト

ユーザーが旅行中に撮影した**複数写真の位置情報（GPS）と時刻情報（撮影日時）**を用いて、\
**自動で「旅の行程（タイムライン）」と「場所ごとのハイライト」を生成し、旅行記録として出力**する。

------------------------------------------------------------------------

## 生成する旅行記録のアウトプット（最小）

-   **旅のサマリー**：旅行日程（開始〜終了）、総移動エリア（代表地点）
-   **時系列タイムライン**：撮影時刻順に並ぶ行程（スポット／滞在推定）
-   **スポット単位のまとめ**：同一地点（近接GPS）＋近接時刻の写真をクラスタ化し、1スポットとして整理
-   **地図表示**：スポットを地図上にプロット（行程線は任意）

※文章の生成は必須ではなく、まずは「並ぶ」「まとまる」「地図に出る」を価値の中心に置く。

------------------------------------------------------------------------

## 解決のアプローチ（アルゴリズム観点の要点）

1.  **データ取得**：写真EXIFから撮影日時・緯度経度を抽出（欠損は除外／補完）\
2.  **時系列整列**：撮影日時でソートし、旅行の流れを作る\
3.  **クラスタリング**：距離閾値（例：100〜300m）＋時間閾値（例：15〜60分）でスポット単位にグルーピング\
4.  **滞在推定**：同一クラスタの滞在時間・写真密度から「立ち寄り/滞在」を推定\
5.  **要約生成**：各スポットの代表写真（ベストショット）と基本情報（時刻範囲、場所名推定）を付与\
6.  **出力**：タイムライン＋地図＋スポットカードとして旅行記録を生成

------------------------------------------------------------------------

## 想定されるメリット（ユーザー価値）

-   **整理コストゼロ**で、旅行後に「見るだけで思い出が戻る」記録が完成する\
-   旅行の流れ（どこ→どこ）が可視化され、**アルバムが"行程"になる**\
-   家族・友人に共有するときに、説明がなくても伝わる（場所と順番が揃っている）

------------------------------------------------------------------------

## 課題・リスク（実装上の懸念）

-   **位置情報欠損**（GPSオフ、屋内、圏外）への対応が必要\
-   **プライバシー配慮**：位置データの取り扱い（オンデバイス処理、明示同意、削除導線）\
-   **誤クラスタ**：近接スポット（駅周辺など）で混ざる／移動中の連写がノイズになる\
-   **地名付与の精度**：逆ジオコーディングの品質・コスト・通信依存

------------------------------------------------------------------------

## MVP（最小実装）提案

-   写真アップロード（複数選択）
-   EXIF抽出（日時・GPS）→時系列整列
-   距離×時間の簡易クラスタリング
-   タイムライン表示（スポットカード＋代表写真＋時刻範囲）
-   地図プロット（スポットピンのみ）
-   位置情報の扱いに関する明示同意＋削除

------------------------------------------------------------------------

## 検証したい主要指標（KPI）

-   旅行後の**自動生成完了率**（アップロード→記録生成まで）
-   生成記録の**閲覧完了率**（最後まで見たか）
-   **修正/分割操作の発生率**（誤クラスタの多さの指標）
-   **共有率**（外部シェア／URL共有）
